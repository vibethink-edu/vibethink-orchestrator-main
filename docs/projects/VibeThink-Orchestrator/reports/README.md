# docs/reports

## Objetivo

Esta carpeta funciona como **repositorio central de reportes, métricas y evidencia documental** para todo el ecosistema VibeThink Orchestrator. Aquí se almacenan reportes de despliegue, KPIs, pendientes, validaciones, resúmenes ejecutivos y cualquier evidencia relevante para auditoría, compliance o seguimiento transversal del proyecto.

## Alcance

- Los reportes aquí documentados pueden ser generales (multi-proyecto) o transversales a varias áreas, módulos o clientes.
- Si un reporte es **específico de un solo proyecto**, debe ubicarse en la carpeta de ese proyecto, por ejemplo:
  - `docs/projects/NombreProyecto/reports/`
- Usa convenciones de nombres claras para identificar el alcance de cada reporte (por ejemplo, `VibeThink-Orchestrator_DEPLOYMENT_FINAL_REPORT.md`).

## Ejemplos de contenido

- Reportes de despliegue global
- Reportes de KPIs por vertical o cliente
- Reportes de pendientes y validaciones
- Casos de uso validados
- Resúmenes ejecutivos
- Evidencia de cumplimiento de objetivos o auditorías

## Buenas prácticas

- Mantén los reportes organizados y actualizados.
- Si un reporte deja de ser transversal y pasa a ser específico de un proyecto, muévelo a la carpeta correspondiente.
- Documenta siempre el alcance y responsable de cada reporte.
