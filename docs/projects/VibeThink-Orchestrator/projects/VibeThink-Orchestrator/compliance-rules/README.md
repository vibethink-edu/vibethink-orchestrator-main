# Políticas y Reglas de Compliance (`compliance-rules`)

---

## Propósito

Esta carpeta centraliza y versiona todas las políticas, reglas y convenciones de compliance obligatorias para VibeThink Orchestrator.

- Cada política se documenta en un archivo independiente para facilitar su mantenimiento, auditoría y referencia cruzada.
- Este índice debe mantenerse actualizado y servir como punto de entrada para cualquier consulta o auditoría.
- **Es obligatorio que toda política transversal de compliance esté aquí y sea referenciada desde las áreas técnicas y de negocio.**

---

## Índice de políticas

- [Política de privacidad y protección de datos](./PRIVACY_POLICY.md): Cumplimiento de GDPR, LOPD y otras normativas de privacidad.
- [Política de retención y borrado de datos](./DATA_RETENTION_POLICY.md): Reglas para la conservación y eliminación segura de datos.
- [Política de gestión de consentimientos](./CONSENT_MANAGEMENT_POLICY.md): Procedimientos para la obtención y gestión de consentimientos de usuarios.
- [Política de cumplimiento normativo sectorial](./SECTOR_COMPLIANCE_POLICY.md): Adaptación a normativas específicas de cada industria o país.

> Agrega aquí cualquier nueva política relevante para el compliance transversal del proyecto.

---

## Patrón replicable

> Se recomienda que cualquier nueva política de compliance transversal se documente aquí, siguiendo el esquema de archivos independientes e índice centralizado. Es obligatorio referenciar desde cualquier área que aplique la política.
