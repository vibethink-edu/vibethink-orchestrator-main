# Políticas y Reglas de Seguridad (`security-rules`)

---

## Propósito

Esta carpeta centraliza y versiona todas las políticas, reglas y convenciones de seguridad obligatorias para VibeThink Orchestrator.

- Cada política se documenta en un archivo independiente para facilitar su mantenimiento, auditoría y referencia cruzada.
- Este índice debe mantenerse actualizado y servir como punto de entrada para cualquier consulta o auditoría.
- **Es obligatorio que toda política transversal de seguridad esté aquí y sea referenciada desde las áreas técnicas y de negocio.**

---

## Índice de políticas

- [Política de control de acceso](./ACCESS_CONTROL_POLICY.md): Reglas y procedimientos para la gestión de accesos y permisos.
- [Política de cifrado](./ENCRYPTION_POLICY.md): Estándares y procedimientos para cifrado de datos en tránsito y en reposo.
- [Política de respuesta a incidentes](./INCIDENT_RESPONSE_POLICY.md): Protocolo para la gestión y reporte de incidentes de seguridad.
- [Política de gestión de vulnerabilidades](./VULNERABILITY_MANAGEMENT_POLICY.md): Proceso para identificar, evaluar y mitigar vulnerabilidades.

> Agrega aquí cualquier nueva política relevante para la seguridad transversal del proyecto.

---

## Patrón replicable

> Se recomienda que cualquier nueva política de seguridad transversal se documente aquí, siguiendo el esquema de archivos independientes e índice centralizado. Es obligatorio referenciar desde cualquier área que aplique la política.
