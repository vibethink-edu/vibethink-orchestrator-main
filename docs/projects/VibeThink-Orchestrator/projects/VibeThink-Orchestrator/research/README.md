# research (VibeThink-Orchestrator)

## Objetivo

Esta carpeta centraliza la **investigación y análisis técnico de software, frameworks o productos externos** relevantes para el proyecto VibeThink Orchestrator. Aquí se documentan evaluaciones, benchmarking, planes de adaptación y entregables de análisis que pueden inspirar, complementar o integrarse a este proyecto.

## Alcance
- Contiene análisis específicos para VibeThink Orchestrator.
- Si en el futuro se requiere investigación transversal, debe ubicarse en `docs/research/`.
- Se recomienda mantener la trazabilidad de responsables, fechas y estado de cada análisis.

## Buenas prácticas
- Documenta siempre el propósito, alcance y responsables de cada investigación.
- Estructura los entregables en subcarpetas temáticas (código, docs, configuraciones, reportes).
- Incluye un plan de análisis y los pasos seguidos para facilitar la reproducibilidad y el aprendizaje organizacional.
- Si se descargan archivos externos (zips, ejemplos), justifica su inclusión y elimina lo innecesario tras el análisis.

## Ejemplo de estructura
```
docs/projects/VibeThink-Orchestrator/research/
├── 01-nombre-app/           # Código fuente o ejemplo
├── 02-nombre-docs/          # Documentación
├── 03-nombre-config/        # Configuración/deployment
├── 04-analysis-reports/     # Reportes y entregables
└── README.md                # Descripción y plan de análisis
```

---

El objetivo es facilitar la toma de decisiones técnicas, la integración de buenas prácticas externas y la mejora continua de VibeThink Orchestrator.
