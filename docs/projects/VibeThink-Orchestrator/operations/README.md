# docs/operations

## Objetivo

Esta carpeta centraliza la **documentación técnica y operativa** para la gestión, monitoreo, backup, observabilidad y mantenimiento de la plataforma VibeThink Orchestrator. Aquí se encuentran guías, procedimientos, registros y configuraciones relevantes para la operación y salud del sistema.

## Alcance
- Documentación transversal a toda la plataforma (no específica de un solo microservicio o módulo).
- Incluye guías de monitoreo, backup, observabilidad, auditoría, estado de la base de datos y procedimientos de mantenimiento.
- Consultada por DevOps, SRE, administradores de base de datos y responsables de operaciones.

## Buenas prácticas
- Mantén los documentos organizados por tema o sistema (ej: monitoreo, backup, observabilidad).
- Si un procedimiento es específico de un solo proyecto, muévelo a la carpeta de ese proyecto (ej: `docs/projects/NombreProyecto/operations/`).
- Archiva en `repo-archive/docs/operations/` cualquier documento legacy u obsoleto.
- Actualiza este README si se agregan nuevas áreas o cambia el alcance.

## Ejemplo de contenido
- Guías de revisión y auditoría de base de datos
- Procedimientos de backup y versionado
- Guías de monitoreo y observabilidad
- Registros de operaciones críticas

---

El objetivo es facilitar la operación segura, eficiente y auditable de la plataforma, centralizando el conocimiento operativo clave.
