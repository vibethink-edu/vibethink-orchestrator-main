# Runtime Canon - Amendments Log

This log tracks all proposed and decided amendments to the Runtime Layer Canon.

## How to Submit an Amendment

All proposals MUST follow the process defined in `RUNTIME_CANON_AMENDMENT_PROCESS_V1.md`. This log only tracks amendments that have reached a final decision (Approved or Rejected).

Links in the log MUST be relative paths to internal PRs or supporting documentation within this repository.

### Type Convention Legend
- **A**: Clarification - No semantic change, improves understanding.
- **B**: Correction - Fixes an error or inconsistency without changing intent.
- **C**: Additive - Adds a non-breaking clarification, rule, or extension (subject to the Amendment Process).
- **D**: Breaking/Deprecation - A breaking change or deprecation of an existing primitive.
