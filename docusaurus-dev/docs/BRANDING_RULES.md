# Reglas de Branding

## Producto vs Metodología
- Producto: siempre «VibeThink Orchestrator»
- Metodología: siempre «VThink 1.0»

## Uso correcto
- Usa «VibeThink Orchestrator» al referirte a la plataforma, dashboards, apps y documentación de producto.
- Usa «VThink 1.0» al referirte a procesos, estándares, cumplimiento y metodología.
- No uses «VThink» para referirte al producto.
- Evita abreviaciones ambiguas como «VT Orchestrator».

## Ejemplos correctos
- «Dashboard de VibeThink Orchestrator»
- «Desarrollado siguiendo la metodología VThink 1.0»

## Ejemplos incorrectos
- «Bienvenido a VThink» (producto mal nombrado)
- «VibeThink 1.0 methodology» (metodología mal nombrada)

## Enforzamiento para IAs
- Cualquier IA (Claude, OpenAI, Gemini, Grok, etc.) debe aplicar estas reglas en commits, PRs y documentación.
- Si se detecta uso incorrecto, corregir automáticamente a:
  - Producto: «VibeThink Orchestrator»
  - Metodología: «VThink 1.0»
